🙋‍♂️ 用户角色
(安全团队/研发团队)
🙋‍♂️ 研发触发/安全专项
  • 【触发行为】执行日常 Git Push [cite: 22, 44]、创建/更新 Merge Request [cite: 44]。
  • 【安全审计】安全团队可按需通过 API 手动提交执行专项审计任务 [cite: 22, 44]。
⏳ 离线静默
  • 【无摩擦体验】无需研发团队干预，不改变日常工作流程 [cite: 42]。
⏳ 离线静默
  • 后台 Agent 多机协作中，安全和研发团队保持离线静默 [cite: 22]。
⏳ 动态等待
  • 系统在此阶段排除“幻觉”误报 [cite: 108]，人类无需介入复核 [cite: 23]。
⏳ 方案等待
  • LLM 后台自动生成并排序修复 diff 候选补丁 [cite: 22, 31]。
⏳ 回归等待
  • 等待系统自动化运行原 PoC 变体与单元测试套件 [cite: 74, 75]。
🙋‍♂️ 关键人工活动
  • 【多级门控】CVSS 7.0-10.0（高危/严重）缺陷生成草稿，强力截流 。
  • 【协同联审】强制触发安全团队 + 研发 Lead 双重人工审核机制 。
🙋‍♂️ 关键人工活动
  • 【消费漏洞】研发 Lead/团队 在 IDE 插件或 Web 看板查看验证报告与 Diff 对比 [cite: 22, 33]。
  • 【合并决策】中危以上（CVSS ≥ 4.0）手动一键合并 PR ；低危自动合并无需人工 。
⏳ 验证闭环
  • PR 合并后，等待平台最终判定漏洞根因是否被彻底消除 [cite: 38]。
💻 AI 平台系统
(前端/看板/接口)
⚙️ 接入层响应
  • 通过 Git Hook / CI/CD 插件 捕获代码变更并验证托管授权 [cite: 22, 44]。
⚙️ 实时推流
  • Web 看板利用 WebSocket 实时渲染静态扫描状态和初筛缺陷列表 [cite: 23]。
⚙️ 编排看板
  • 提供 Agent 推理协同看板，结构化记录分析日志 [cite: 23]。
⚙️ 验证详情
  • 平台结构化承载并展示 PoC 脚本、数据流路径与危害说明 [cite: 23]。
⚙️ 补丁对比
  • 界面呈现多候选 diff 补丁，并附置信度、侵入性排序标识 [cite: 22, 69]。
⚙️ 报告生成
  • 呈现三重验证数据（原 PoC 重跑结果、性能基准变动和二次 SAST 扫描） [cite: 73]。
⚙️ 阻断门控
  • 针对高危缺陷下发人工审核工单，冻结 PR 自动合入权限，强制等待审批 。
⚙️ 通知消费
  • 将验证通过的补丁自动封装为 PR，通过 Jira / Slack / 邮件 推送给对应团队 [cite: 22, 33]。
⚙️ 状态流转
  • 跟踪漏洞状态机（图3），重扫通过后正式将状态流转为终态【已关闭】 [cite: 81, 82]。
🤖 AI 核心大脑
(分析引擎/多Agent/流水线)
⚙️ 代码预处理
  • 拉取代码，自动构建 AST（抽象语法树）并提取跨文件函数调用图 (Call Graph) [cite: 47, 48]。
⚙️ 混合双引擎
  • 运行 CodeQL 做语义级污点追踪，并发运行 Semgrep 进行规则级匹配 [cite: 54]。
⚙️ 多 Agent 协同
  • Planner Agent 调度：Researcher 查历史 CVE [cite: 59] → Analyst 评估可达路径与根因 [cite: 59] → Exploiter 构造可复现的最小 PoC 代码 [cite: 17, 59]。
⚙️ 沙箱隔离执行
  • 在 Docker + gVisor 完全隔离沙箱 中动态运行 PoC 触发 crash，并计算 CVSS 评分 [cite: 22, 66]。
⚙️ Patcher 驱动
  • 由 Patcher Agent 调用 LLM（云端 Claude / 私有部署 Qwen/DeepSeek-Coder）多方案生成 diff 补丁 [cite: 59, 89]。
⚙️ 三重验证闭环
  • 1. 回归：重跑原 PoC 变体 [cite: 74]；2. 副作用：跑单测，性能劣化 ≤ 5% [cite: 75]；3. 二次扫描：防引入新 CWE [cite: 76]。
  • 【回路设计】若验证失败支持最多 3 次自动重试；超限则升级为人工处理 [cite: 36, 37]。
⚙️ 守候响应
  • 挂起当前自动化流程，保持对人工审核判定结果（通过/驳回）的监听 [cite: 23, 71]。
⚙️ 自动提交
  • 低危缺陷根据策略自动提交并合入主干 ；中高危在收到人类 Approve 指令后自动执行 PR 合并 [cite: 22, 34]。
⚙️ 终审重扫
  • 代码合并后自动触发全量扫描，检测补丁修复因果链，确保全产品线安全基线稳固 [cite: 13, 38]。
